# 2026-07-21 — Sprint Log

The whole project — CRUD API, live Firebase sync, a 3D globe, an installable PWA, two provider-variant branches, an SEO/AEO layer, a mobile rework, and analytics — was built in one sitting: 25 commits across `main`, `feat/google-only`, and `feat/seo-aeo`, 09:52 to 15:51.

## Timeline

### 09:52 — Core build
`209cd0e` Implement User CRUD API with location enrichment + React UI. The whole shape of the system landed in one commit: the repository facade, the single OWM lookup, the client-trust boundary, and the error model. See [[ADR-0001-db-facade]], [[ADR-0002-owm-single-call]], [[ADR-0004-trust-boundary]], [[ADR-0005-error-model]].

### 10:07–10:58 — Firebase + frontend platform
`aec4fae` Firebase configurations, then `b802117` migrated the frontend to Vite + React with ReactFire as a real dependency (see [[ADR-0006-frontend-live-sync]]), `6f2a94f` wired the live Firebase RTDB project with rules and ADC (see [[ADR-0007-firebase-adc-rules]]), and `4095a84` added Vitest code coverage reaching ~92% of `src/` (see [[ADR-0008-testing-strategy]]). Four commits, four ADRs' worth of foundation, inside an hour.

### 11:08–11:27 — Docs and branding pass
`8c115b7` added the project purpose and a "Decisions Explained" section to the README, `e6c6f81` applied RentRedi branding with a light/dark toggle, and `8b5f2c7` added the "Dan Coyle's Assessment" title to the top bar — making the take-home legible as a take-home before pushing further.

### 11:51 — DB tooling
`6d9e8c8` added an RTDB setup script that deploys rules, seeds data, and verifies the connection — turning "wire up Firebase" from a manual checklist into one command.

### 12:39–13:21 — Globe and installability
`4b2b8b2` shipped an interactive three-globe view with day/night shading, tooltips, and country support; `71b9882` made the app an installable PWA with offline support; `b8ba668` added an offline indicator with reactive connectivity handling. The visual centerpiece and the offline story landed back to back.

### 13:22–13:35 — Branch experiments: SEO/AEO and Places autocomplete
`2c556e9` on `feat/seo-aeo` added robots.txt, sitemap, JSON-LD, and Open Graph/noscript for answer-engine optimization (see [[SEO and AEO]]). In parallel, `ed927a1` added single-input city/ZIP autocomplete via Google Places with perf fixes (see [[Provider Variants]]), `3c7e64d` reverse-geocoded coordinates to show a ZIP for city results, and `2c0d322` swapped in plain ZIP + country inputs when the PWA is offline, since Places autocomplete needs a live connection.

### 13:57–14:53 — Railway build detours
Two build-only fixes to get the Railway deploy target green — covered under **Detours & fixes** below.

### 15:13–15:51 — Provider finalization, mobile layout, analytics
`ae534b7` removed Google Places to make `main` OpenWeatherMap-only (see [[Provider Variants]]), `bec547f` reworked the UI into a responsive mobile layout (modal form, globe-first, horizontal list), and `fc538b4` made the mirror-image call on `feat/google-only` — removing OWM to make that branch Google-Places-only (see [[Provider Variants]]). `adaecf0` and `19d3da8` merged the responsive layout into both branches. The day closed with `24d77df` adding Google Analytics 4 (event + goal tracking) to `main`, and `e85db25` porting the same GA4 work to `feat/google-only`.

## Decisions that came out of today

- [[ADR-0001-db-facade]] — in-memory default, Firebase behind one repository facade.
- [[ADR-0002-owm-single-call]] — one OpenWeatherMap call for coordinates and timezone.
- [[ADR-0003-refetch-on-change]] — refetch location only when the ZIP or country changes.
- [[ADR-0004-trust-boundary]] — `.strict()` schemas reject client-supplied derived fields.
- [[ADR-0005-error-model]] — typed errors funneled through one central handler.
- [[ADR-0006-frontend-live-sync]] — Vite + ReactFire live reads, with a polling fallback.
- [[ADR-0007-firebase-adc-rules]] — ADC over key files, least-privilege RTDB rules.
- [[ADR-0008-testing-strategy]] — hermetic tests by construction, ~92% coverage.

## Detours & fixes

- **`vite-plugin-pwa` 0.20 → 1.3** (`7a66d6c`, 13:57) — the PWA plugin version in use was breaking the Railway build; upgrading two major versions fixed it.
- **Node >=20, pinned to 22 via `.nvmrc`** (`48c04b8`, 14:53) — Railway's build environment was resolving an older Node than the toolchain needed, so the engine requirement was pinned explicitly.

Both were build-pipeline problems, not application bugs — neither changed runtime behavior.

## State at end of day

`main` = OpenWeatherMap-only + interactive globe + installable/offline PWA + GA4 analytics + responsive mobile layout. Two experiment branches parked: `feat/google-only` (the Google-Places-only mirror) and `feat/seo-aeo` (the SEO/AEO layer) — see [[Experiments]].
